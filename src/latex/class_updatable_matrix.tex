\subsection{Detailed Description}
\subsubsection*{template$<$typename Matrix\+N\+D$>$class Updatable\+Matrix$<$ Matrix\+N\+D $>$}

A matrix that can be created in 1, 2, 3, or 4 dimensions with the ability to be updated. 

Updatable matrix -\/ 1,2,3,4 dimensions.

It\textquotesingle{}s a template because it\textquotesingle{}s easier to create, maintain, and debug one class, instead of 4 classes with the same functionality

It\textquotesingle{}s just like a normal matrix (and can be used just like one), but it can also be updated according to rules from an ini-\/file

The read\+From\+Ini\+File function is used to load data into an \hyperlink{class_updatable_matrix}{Updatable\+Matrix} by storing the contents of a .tab file. The .tab file will either list a .plt file to load or a .lst file which in turn has both time steps and .plt files. The update function is called at every time step in the main function and will execute any updates that can be found from these files that match the designated timestep.

Read Update\+Matrix parameters from a string

Will traverse the string and get the entire lines values -\/ starting with parameter q1 and getting values until reaching q4 or running out of values for the line

If a data file (such as .plt) is selected then the contents will be stored into the matrix. If a update-\/file (such as .lst) is selected then no matrix values will be saved, however the update\+\_\+filename field will be saved. Upon every timestep the main file will call \hyperlink{class_updatable_matrix_a57fe16cce1982a1df412fe9a3e1d28ea}{update()} on this matrix, and inside update will look for the update\+\_\+filename field. If present it will update the matrix to the corresponding timestep
